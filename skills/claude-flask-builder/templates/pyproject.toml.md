# Template — pyproject.toml

Path: `pyproject.toml` at the `api/` project root (sibling of `Pipfile`).

Centralizes lint + format + test config. Scaffolder writes this file; CI
and `flask-pattern-reviewer` Stage 1 both invoke `ruff` and `black` with
these settings.

```toml
[build-system]
# WHY no setuptools build: we run as a pipenv app, not a published wheel.
# Keep [build-system] absent (or comment) — pyproject is config-only.

[tool.ruff]
line-length = 100
target-version = "py311"
# WHY these selects (R03 isort, R05 type hints, R10–R16 naming, R20–R22
# security via flake8-bandit): match the reviewer Stage 1 contract.
extend-include = ["*.py"]
extend-exclude = ["migrations/versions"]

[tool.ruff.lint]
select = [
    "E", "F",       # pycodestyle + pyflakes (base)
    "I",            # isort (R03)
    "B",            # flake8-bugbear
    "N",            # pep8-naming → assists R10–R16
    "UP",           # pyupgrade — modern Python idioms
    "ANN",          # flake8-annotations → assists R05
    "S",            # flake8-bandit → assists R20–R22
    "SIM",          # flake8-simplify
    "RUF",          # ruff-specific
]
ignore = [
    "ANN101",  # missing-type-self — Python infers, not actionable
    "ANN102",  # missing-type-cls
    "S101",    # assert in src — pytest uses asserts
    "E501",    # line-too-long — black handles
]

[tool.ruff.lint.per-file-ignores]
"app/tests/**/*.py" = ["S101", "ANN", "S105", "S106"]  # tests: asserts + hardcoded test passwords
"migrations/versions/*.py" = ["ANN", "E501"]            # Alembic autogenerated
# Note: app/utils/Constants.py used to have an N816 override here. After
# Round 8 it became UPPER_SNAKE_CASE module-level strings — N816 does NOT
# apply, so no override needed.

[tool.ruff.lint.isort]
known-first-party = ["app"]
combine-as-imports = true
force-sort-within-sections = true

[tool.black]
line-length = 100
target-version = ["py311"]
extend-exclude = '''
/(
    migrations/versions
)/
'''

[tool.pytest.ini_options]
minversion = "8.0"
testpaths = ["app/tests"]
addopts = [
    "-ra",
    "--strict-markers",
    "--strict-config",
    "--cov=app",
    "--cov-report=term-missing",
    "--cov-fail-under=60",
]
filterwarnings = [
    "error",
    "ignore::DeprecationWarning:flask_seeder.*",
    "ignore::DeprecationWarning:passlib.*",
]
markers = [
    "e2e: full request → DB → response",
    "integration: endpoint with mocked externals",
]

[tool.coverage.run]
branch = true
source = ["app"]
omit = [
    "app/tests/*",
    "app/scheduler_jobs/__init__.py",
    "migrations/*",
]

[tool.coverage.report]
exclude_lines = [
    "pragma: no cover",
    "raise NotImplementedError",
    "if TYPE_CHECKING:",
    "if __name__ == .__main__.:",
]
```

## Checklist (reviewer enforces)
- `[tool.ruff]` line-length = 100 (matches R02).
- `[tool.ruff.lint] select` includes `E,F,I,B,N,UP,ANN,S` (matches the
  reviewer Stage 1 contract).
- `[tool.black]` line-length = 100 (matches ruff).
- `[tool.pytest.ini_options].testpaths = ["app/tests"]` (pytest invoked
  from `api/` cwd per `testing.md`).
- `--cov-fail-under=60` is present (R102).
- `branch = true` under `[tool.coverage.run]` — branch coverage is the
  honest metric; line coverage alone misleads.
